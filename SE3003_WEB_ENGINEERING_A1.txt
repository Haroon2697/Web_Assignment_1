  Frontend Admin Dashboard Web Application



1. Objective

Students are required to design and develop a multi-page admin dashboard web application
using:

    ´éÀ   HTML (structure)
    ´éÀ   CSS (layout, responsiveness, styling)
    ´éÀ   JavaScript (logic, interactivity, data handling)

The application must simulate real-world functionality using mock data and browser storage
(localStorage).

2. General Rules and Constraints

    1. No backend technologies are allowed (no Node.js, PHP, databases, etc.).
    2. All data must be handled using:
            o    JavaScript arrays (mock data)
            o    localStorage (for persistence)
    3. All pages must be connected and navigable.
    4. Code must be modular, readable, and properly structured.
    5. Responsive design is mandatory.
    6. All features must function without page reloads where possible.

3. Authentication Workaround (Mandatory)

Since there is no backend, authentication must be simulated using localStorage.

1. Signup (User Creation)

    ´éÀ   User enters email and password.
    ´éÀ   Data is stored in localStorage (not sent to a server).
    ´éÀ   Save as an object: { email, password }.

Summary: Browser acts as a temporary database.

2. Login (User Verification)

     ´éÀ    Retrieve stored user data from localStorage.
     ´éÀ    Compare entered email & password with stored values.
If match:
     ´éÀ    Mark user as logged in (e.g., loggedIn = true in localStorage).
     ´éÀ    Redirect to dashboard.
If not:
     ´éÀ    Show error: ÔÇ£Invalid email or passwordÔÇØ.
3. Session Simulation

     ´éÀ    Store a flag (e.g., loggedIn) in localStorage after login.
     ´éÀ    Use it to track login state.
Summary: localStorage acts as a fake session.
4. Route Protection

     ´éÀ    On every protected page:
               o    Check loggedIn in localStorage.
If not present:
     ´éÀ    Redirect to login page.
If present:
     ´éÀ    Allow access.
5. Limitations (Important)

     ´éÀ    Not secure
     ´éÀ    Data can be viewed/edited via browser dev tools
     ´éÀ    Only for learning/demo purposes
